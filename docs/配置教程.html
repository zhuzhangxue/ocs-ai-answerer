📖 OCS AI Answerer 配置修改完整教程

这个教程适用于：你想改模型的思考程度（low/medium/high）、切换 Flash/Pro、换 API key、改 temperature 等任何配置。

所有配置都存在 OCS-AI-Answerer-一键包\custom_models.json 这一个文件里（即你解压后的项目根目录），改完必须重启服务才生效。

第 1 步：找到并打开文件

  1. 用记事本 / VSCode / Notepad++ 打开： 
    OCS-AI-Answerer-一键包\custom_models.json
  2. 推荐用 VSCode，有 JSON 高亮和格式校验，更安全。
  3. 文件 不要用 Word/WPS 打开，会破坏编码。

⚠️ 改之前先备份！把文件复制一份到 custom_models.json.bak，改坏了能恢复。

第 2 步：认识文件结构

文件有两块："models"（模型定义）和 "question_type_models"（题型映射）。

2.1 models（模型定义）

每条模型都长这样：

"preset_deepseek_v4_flash": {
  "name": "DeepSeek V4 Flash",
  "provider": "openai",
  "api_key": "sk-你的真实key",
  "base_url": "https://api.deepseek.com",
  "model_name": "deepseek-v4-flash",
  "is_multimodal": false,
  "max_tokens": 500,
  "temperature": 0.1,
  "top_p": 0.95,
  "supports_reasoning": true,
  "reasoning_param_name": "reasoning_effort",
  "reasoning_param_value": "medium",       ← 🔥 改这里调思考程度
  "api_protocol": "openai_compat",
  "enabled": true
}

2.2 当前 9 个模型 ID（记住这些名字）

模型 ID显示名用途
preset_deepseek_v4_flashDeepSeek V4 Flash文字题（快、便宜）
preset_deepseek_v4_proDeepSeek V4 Pro文字题（准、慢）
preset_glm_4v_flashGLM-4.6V-Flash图片题（免费视觉，首选）
注意：免费但可能限流，限流后自动降级
preset_doubao_mini豆包 Mini 2.0图片题文字链（付费，试用额度非永久免费）
preset_doubao_turbo豆包 Turbo 2.1图片题（付费）
preset_doubao_pro豆包 Pro 2.0图片题（付费）
preset_doubao_2_1_pro豆包 Pro 2.1图片题复杂链（付费，最终兜底）
preset_qwen_vl_flashQwen VL Flash图片题（付费轻量级视觉）
preset_qwen_3_7_plusQwen3.7-Plus图片题复杂链（付费强力视觉）

2.3 question_type_models（题型映射）

"question_type_models": {
  "single":     { "models": ["preset_deepseek_v4_flash", "preset_deepseek_v4_pro"] },
  "multiple":   { "models": ["preset_deepseek_v4_pro", "preset_deepseek_v4_flash"] },
  "judgement":  { "models": ["preset_deepseek_v4_flash", "preset_deepseek_v4_pro"] },
  "completion": { "models": ["preset_deepseek_v4_flash", "preset_deepseek_v4_pro"] },
  "image":      { "models": ["preset_glm_4v_flash", "preset_doubao_mini", "preset_qwen_vl_flash", "preset_qwen_3_7_plus", "preset_doubao_2_1_pro"] }
}
  • single = 单选 / multiple = 多选 / judgement = 判断 / completion = 填空
  • image = 图片题云端降级链：本地 OCR 失败后，按 GLM → 豆包 Mini → Qwen VL Flash → Qwen3.7-Plus → 豆包 Pro 2.1 顺序调用
  • 每个题型 "models" 数组里 第一个是主用，后面的都是 fallback（主用失败自动降级）
  • 本地 OCR 始终最先跑，达标则完全不调用云端视觉模型，零成本

第 3 步：常见修改场景（复制即用）

场景 1：把某个模型的思考程度调成 high（深度思考）

比如想让 preset_deepseek_v4_flash 用 high 思考：

  1. 搜 preset_deepseek_v4_flash
  2. 找 "reasoning_param_value": "medium"
  3. 改成 "reasoning_param_value": "high"
"reasoning_param_name": "reasoning_effort",
"reasoning_param_value": "high"   ← 改这里

三档说明：

档位耗时适用
low3-5 秒选择/判断等简单题
medium5-15 秒填空/一般计算（默认）
high20-60 秒证明题/复杂推导/求导积分

场景 2：把单选/填空从 Flash 切到 Pro

想让"单选题"和"填空题"优先用 Pro（更准但慢）：

  1. 搜 "question_type_models"
  2. 把 "single" 和 "completion" 里的顺序调换
"single": {
  "models": [
    "preset_deepseek_v4_pro",      ← 主用 Pro（准确）
    "preset_deepseek_v4_flash"     ← 失败降级到 Flash
  ]
},
"completion": {
  "models": [
    "preset_deepseek_v4_pro",      ← 主用 Pro
    "preset_deepseek_v4_flash"     ← 失败降级
  ]
}

💡 想让所有题都用 Pro？ 把每个题型的 "models" 数组第一个都改成 preset_deepseek_v4_pro。

场景 3：调整输出长度（max_tokens）

当前 preset_deepseek_v4_flash 的 max_tokens 只有 500——答问答题时可能不够（返回截断或空）。改成 4096：

"preset_deepseek_v4_flash": {
  ...
  "max_tokens": 4096,    ← 改这里
  ...
}

问答题（【其它】类型：求函数单调区间、证明题等）建议至少 4096。

场景 3.5：调整 temperature（AI 随机性）

值效果适用
0.0每次答案一样填空/判断（推荐）
0.1（默认）几乎一样，偶尔不同通用
0.7比较发散简答题/创作
1.0+很发散一般不用
"temperature": 0.0    ← 填空题想稳定就改 0

场景 4：换 API Key

比如 DS 余额不足要换新 key：

"preset_deepseek_v4_flash": {
  "api_key": "sk-你的新key",   ← 改这里
  ...
}

场景 5：换底层模型（API 升级/换平台）

比如想从 DeepSeek V4 切到 V3.5，或者换到 GPT：

"preset_deepseek_v4_flash": {
  "name": "DeepSeek V3.5",     ← 显示名（随便起）
  "base_url": "https://api.deepseek.com",   ← API 地址
  "model_name": "deepseek-v3.5-flash",      ← 实际模型 ID
  "api_key": "sk-xxx",
  ...
}

场景 6：临时禁用某个模型

比如豆包平台今天抽风，先禁用豆包只用 DS：

"preset_doubao_mini": {
  "enabled": false,   ← 改成 false
  ...
}

或者从题型映射里把豆包删了：

"image": {
  "models": [
    "preset_glm_4v_flash",
    "preset_doubao_turbo",        ← 主用还在的豆包
    "preset_qwen_vl_flash"        ← 兜底
    // 删掉 preset_doubao_mini
  ]
}

第 4 步：保存 + 重启服务

⚠️ 改完必须重启！服务在启动时读这个文件，运行中改 不会生效。

重启步骤：

  1. 关掉旧服务窗口（点 X 关闭那个黑色 cmd 窗口）
  2. 打开新窗口，执行（把路径换成你的项目实际位置）： 
    cd /d C:\你的路径\OCS-AI-Answerer-一键包
.venv\Scripts\python.exe ocs_ai_answerer_advanced.py
  3. 看到 ✅ 服务启动成功！ 就 OK 了

验证改成功：

  1. 浏览器打开 http://127.0.0.1:5000/api/health
  2. 看到 "enabled_model_count": 7 说明服务正常
  3. 用 OCS 触发一道题，看 CSV 日志（ocs_answers_log.csv）的"模型"列

第 5 步：常见错误和排查

❌ 改完服务启动报错 JSONDecodeError

原因：JSON 格式坏了（漏了逗号、引号不对等）

修复：

  1. VSCode 会标红，按提示补
  2. 实在不行就把 custom_models.json.bak 备份还原
  3. 在线工具 https://jsonlint.com 校验格式

❌ 改完没生效

原因：忘了重启服务

修复：按第 4 步重启

❌ 改完 AI 答空 / 报错

原因 1：api_key 填错

原因 2：model_name 写错（注意空格、大小写）

原因 3：base_url 写错（要带 https://，不要带尾部 /）

排查方法：看后端日志里的"HTTP Request"行，看具体 API 返回什么错。

附录：完整 JSON 字段速查

字段类型作用常用值
name字符串面板显示名任意
provider字符串识别用openai / anthropic 等
api_key字符串API 密钥你的 key
base_url字符串API 端点https://api.deepseek.com
model_name字符串实际模型 IDdeepseek-v4-flash
is_multimodal布尔是否支持图片true/false
max_tokens整数最大输出长度500 / 4096 / 8192
temperature浮点随机性 0-20.1 (答题) / 0.7 (创作)
top_p浮点采样阈值 0-10.95
supports_reasoning布尔是否支持思考true/false
reasoning_param_name字符串API 思考参数名reasoning_effort / enable_thinking
reasoning_param_value字符串思考档位low/medium/high 或 true/false
api_protocol字符串API 协议openai_compat / responses
enabled布尔是否启用true/false
