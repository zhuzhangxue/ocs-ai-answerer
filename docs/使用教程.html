🔑 第二阶段：获取 API Key 3 分钟
💡 什么是 API Key？它是你调用 AI 大模型的"通行证"。本项目用两个：DeepSeek（推理大脑）+ 豆包（视觉眼睛）。
1

注册 DeepSeek 并获取 Key 必需

DeepSeek 负责"思考答题"，是整个项目的大脑。

打开 DeepSeek 平台 →
  1. 点击右上角「注册」(支持手机号/微信)
  2. 登录后点击左栏「API Keys」
  3. 点击「创建 API Key」→ 名字随便填(如 "OCS")
  4. 立即复制显示的 key（形如 sk-xxxxxxxxxxxxxxxx）
⚠️ Key 只显示一次！必须立刻复制保存到记事本
💰 费用: 充值 1 元就够用很久(V4 极便宜,具体价格以 DeepSeek 价格页 为准)
2

注册火山方舟并获取豆包 Key 必需

豆包负责"看图认字"(OCR),用火山方舟平台调用。

打开火山方舟 →
  1. 用抖音/手机号注册登录
  2. 点击左侧栏目「推理」
  3. 进入「在线推理」页面
  4. 点击「快速入门」
  5. 在快速入门页点击「创建 API Key」
  6. 勾选所有模型权限 → 创建
  7. 立即复制Key(形如 ark-xxxxxxxx-xxxxxxxx-xxxxxxxx)
💰 费用: 豆包 seed 系列有大量免费额度,基本不花钱
3

在豆包平台开通模型 必需

开通模型后才能调用,操作是免费的。提供两种方式,选一种即可：

方式 A：快速入门入口(推荐)
  1. 还是刚才的「快速入门」页面
  2. 往下翻,找到「Step2 快速接入测试」模块
  3. 在模型下拉框里选 doubao-seed-2-0-mini
  4. 点旁边的「开通」按钮
  5. (可选)再选 doubao-seed-2-1-turbo 和 doubao-seed-2-1-pro 也开通
方式 B：模型广场入口
  1. 顶部菜单点「模型广场」
  2. 搜索 doubao-seed 找到 seed 系列
  3. 点右上角「API 接入」按钮
  4. 会跳到「Step2 快速接入测试」页面
  5. 在模型下拉里选 doubao-seed-2-0-mini → 开通
  6. 同样方法开通 turbo 和 pro
✅ 现在你手上有 2 个 key: 一个 sk- 开头,一个 ark- 开头
⚡ 第三阶段：填 Key + 双击启动 2 分钟
1

填入 keys.txt 关键

在 OCS-AI-Answerer-一键包 文件夹里,找到 keys.txt

右键 → 「打开方式」→ 「记事本」

把里面的占位符替换成你的真实 key：

# 改成你的真实 key（删除 <> 部分,只保留 key 本身）
DEEPSEEK_API_KEY=sk-你刚才复制的 DeepSeek key
DOUBAO_API_KEY=ark-你刚才复制的火山方舟 key

按 Ctrl + S 保存,关闭记事本。

⚠️ 格式必须保持 KEY=值 中间不要有空格、不要加引号、不要有空行
📷 keys.txt 正确示例(实际部署时,key 会变成真实字符串)
DEEPSEEK_API_KEY=sk-你的真实key
DOUBAO_API_KEY=ark-你的真实key

📋 第五阶段：导入题库配置 1 分钟

打开 ocs_config.json

在 OCS-AI-Answerer-一键包 文件夹里,找到 ocs_config.json

右键 → 「打开方式」→ 「记事本」

按 Ctrl + A 全选,按 Ctrl + C 复制

2

在 OCS 悬浮窗导入 必需

在 OCS 悬浮窗里操作：

  1. 点「设置」(齿轮图标)
  2. 找到「题库配置」→ 点「配置」按钮
  3. 把刚才复制的内容粘贴进去
  4. 点「解析并保存」
✅ 底部出现「AI 智能答题」字样 = 导入成功
⚠️ 如果报「Spread syntax requires...」错,说明 OCS 版本太老,需要更新 OCS 脚本到最新
3

(可选)调整题库设置

在 OCS 设置里建议调整：

  • 答案匹配模式: 改成「精确匹配」(避免 "3" 误匹配 "-3")
  • 搜题最大耗时: 调到 180 秒(图片题可能需要 30-60 秒,留足余量)

❓ 常见问题 FAQ
Q1: 双击 bat 后,cmd 窗口一闪而过?
可能是编码问题。右键 bat → 编辑,把第 2 行 chcp 65001 改成 chcp 936,或者用 Windows Terminal 运行。
Q2: 步骤 1 提示「未检测到 Python」?
Python 没装或者没勾「Add to PATH」。重装 Python 时务必勾上 Add Python to PATH 选项。
Q3: 步骤 3 依赖安装失败?
国内网络问题。先在 cmd 运行：
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
再重新运行 bat。
Q4: 步骤 4 检测不到 key?
检查 keys.txt 格式：
① 不能有空格（KEY=值，不是 KEY = 值）
② DeepSeek key 必须 sk- 开头，豆包 key 必须 ark- 开头
③ 不能有引号
④ 不能用中文标点
Q5: 浏览器打开 127.0.0.1:5000 提示「无法连接」?
服务没起来。在 cmd 窗口里看有没有错误,或者检查 5000 端口是否被别的程序占用。
Q6: OCS 导入题库报「Spread syntax」错?
OCS 脚本版本太老(低于 4.7.21)。去 OCS 官网更新脚本到最新版。
Q7: 答题时显示「此题未完成」?
常见原因(按概率排序)：
① OCS 端用「相似匹配」→ 改成「精确匹配」
② 答案格式不匹配 → 单选答字母而非文字
③ 图片题豆包调用失败 → 检查 .env 里 key 是否有效
④ 超时 → OCS 设置里把题库耗时调到 60s
Q8: 怎么添加新模型(如 Kimi)？
打开 docs/添加新模型教程.html 里有完整步骤。或者直接告诉我「加 Kimi, key 是 xxx」,我帮你加。
Q9: 换电脑后怎么迁移?
压缩整个 OCS-AI-Answerer-一键包 文件夹(含 venv) 拷到新电脑,改 keys.txt,双击 bat 即可。venv 目录可以删掉重新创建(小电脑空间不够时)。
Q10: 想换个模型(如 Flash 改 Pro)？
打开 docs/配置教程.html 里有详细步骤。或者告诉我「单选切换到 Pro」,我帮你改。
