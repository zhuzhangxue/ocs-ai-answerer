想接入 Kimi K3、Qwen3、GPT-4o、Claude、Gemini 等任何模型,都可以加进来用。

方式 2: 自己改 custom_models.json

3 步搞定

第 1 步: 添加模型定义

打开 custom_models.json,找到 "models" 字典,在末尾加一条 (逗号别忘):

"preset_kimi_k3": {
  "name": "Kimi K3",
  "provider": "openai",
  "api_key": "sk-你的Kimi密钥",
  "base_url": "https://api.moonshot.cn/v1",
  "model_name": "moonshot-v1-128k",
  "is_multimodal": false,
  "max_tokens": 4096,
  "temperature": 0.1,
  "top_p": 0.95,
  "supports_reasoning": true,
  "reasoning_param_name": "reasoning_effort",
  "reasoning_param_value": "medium",
  "api_protocol": "openai_compat",
  "enabled": true,
  "is_builtin": false
}

第 2 步: 加到题型映射

在同一文件,找到 "question_type_models",把你新模型加到想要的题型:

"question_type_models": {
  "single": {
    "models": [
      "preset_kimi_k3",              ← 你刚加的
      "preset_deepseek_v4_flash",
      "preset_deepseek_v4_pro"
    ]
  },
  "multiple": {
    "models": [
      "preset_kimi_k3",
      "preset_deepseek_v4_pro"
    ]
  }
}

放在数组第一个 = 主用,后面的 = 降级备用。

第 3 步: 重启服务

netstat -ano | findstr :5000
taskkill /F /PID <上面的PID>
一键启动.bat

或者直接关闭旧窗口,跑 一键启动.bat。

完整工作流

添加新模型的标准流程:

  1. 在平台开账号、拿 API key
  2. 在平台控制台开通/订阅你要的模型
  3. (可选) 小金额充值 (一般 5-10 元够用很久)
  4. 用 curl 或平台 playground 测一下 API 通了
  5. 在 custom_models.json 加模型
  6. 在 question_type_models 加映射
  7. 重启服务
  8. 用 OCS 触发一道题,看 CSV 日志的"模型"列验证

遇到问题?

把 错误信息 + 你想加的模型 发给我,我帮你诊断。

常见错误:

  • 401 Unauthorized → API key 错或没付费
  • 404 Not Found → base_url 或 model_name 写错
  • 400 Bad Request → 参数不兼容 (比如思考参数名不对)
  • 404 InvalidEndpointOrModel.NotFound → 平台没开通这个模型

截图或粘贴错误信息,告诉我,我帮你看。
