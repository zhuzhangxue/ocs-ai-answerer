# 安全政策

## 支持的版本

| 版本 | 支持状态 |
|---|---|
| 1.0.x | ✅ 积极维护 |
| < 1.0 | ❌ 不再支持 |

## 报告漏洞

我们非常重视安全问题。如果发现安全漏洞，请**不要在公开 Issue 里报告**。

请通过以下方式私下联系：
- 邮件：[your-real-email@example.com](mailto:your-real-email@example.com)（请替换为你的邮箱）
- 标题：`[SECURITY] OCS AI Answerer 漏洞报告`

### 报告应包含

1. 漏洞描述
2. 复现步骤
3. 潜在影响
4. 建议修复方案（可选）

我们会在 48 小时内回复。

## 安全最佳实践

部署本项目时，请：

- ❌ **不要** 把 `.env` 或 `keys.txt` 提交到 Git
- ❌ **不要** 把真实 API key 写在 Issue / PR / 讨论里
- ✅ **推荐** 用 GitHub Secrets 存 CI 用的 key
- ✅ **推荐** 定期轮换 API key
- ✅ **推荐** 在生产环境用 HTTPS（虽然本项目是本地服务）

## 已知的非问题项

为了透明，以下"漏洞"**不是真正的安全问题**：

- 本地服务默认只监听 `127.0.0.1`，外部无法访问
- `/api/answer` 不需要鉴权（仅本地访问）
- `/config_legacy` 显示配置但不含明文 key（被脱敏）

如果你有疑虑或建议，欢迎邮件联系。
